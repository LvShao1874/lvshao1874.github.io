重温计算机网络

发表于 2019-11-05 更新于 2019-11-13

开头废话

时隔3年,依然记得大二那年学的计算机网络.教材书用的是计算机网络(第五版).
当时对计算机网络有了一些认识.学习网络的过程个人理解为送快递的过程有相似之处..

计算机网络协议层级

数据链路

  1. 数据链路是什么:
    数据链路,指OSI参考模型中的数据链路层,有时也指具体的以太网,无线局域网等通信手段.
    数据链路层的协议定义了通过通信媒介互连的设备之间传输的规范.实际上,各个设备之间的数据传输时,数据链路层和物理层都是必不可少的.
    因为计算机是以二进制0、1来表示信息.然而在实际的通信媒介的信号却是电压的高低,光的闪灭,以及电波的强弱等信号.把这些信号与二进制的0、1进行转换便是物理层主要职责.
    数据链路层处理的数据则是在物理层的基础上将0、1序列集合为一个叫做帧的块,然后再进行传输.

  2. MAC地址
    MAC地址用于识别数据链路中互连的节点.MAC地址长48比特,结构如下所示.MAC地址基本是全球唯一的(不总是唯一的,只要在同一链路上便不会有问题).

IP协议

  1. ip协议是什么
    IP(Internet Protocol),即网际协议.主要负责将数据包发送给最终目标计算机.IP协议能够让世界上任何两台计算机之间进行通信.
    1. 协议层级:
      位于网络层,与数据链路层的关系:数据链路提供2个直连设备之间的通信功能,
      而网络层的IP协议则负责让没有直连的网络中的计算机进行通信.
    2. IP属于面向无连接型:
      主要有2点原因:1.简化 2.提速.
      面向无连接可以保证基于IP协议的通信更新轻量,保持连接会花费更多的处理时间.降低通信速度.
      而且通过网络协议的规划,每一层级遵循单一职责(避免实现复杂).在应用层有相应的TCP协议来实现有连接通信.

  1. IP地址的分类
    IP地址有由32位比特(0,1)(即4字节)来标识,用十进制来表示则为xxx.xxx.xxx.xxx,其中xxx的范围为0~255.
    第四个xxx中0和255有特殊用途,不作为ip地址.
    IP地址分为4个级别,A,B,C,D类.根据IP地址中第一位到第四位的比特列对齐网络标识和主机标识进行区分.

    由于A,B,C,D分类方式导致IP地址存在大量浪费(因为A类的IP地址,该网段下可以有16777214个主机号,然而获得该网段的使用者并无法使用如此多的IP),
    所以借助子网掩码,可以将任一IP所属网段的网络进一步细分为更多的子网络,
    达到IP地址的高效利用.后面又使用CIDR(无类型域间选路),对IP进行更加任意的划分),例如203.183.224.0/23,将203.183.224.1到203.183.225.254这2个网段(原本均为C类地址)合并为一个网络,
    新的网络容纳主机510台(2^9-2).比C类地址原本的254主机(2^8-2)相比,可以容纳更多的主机.

    然而使用上面的技术并不能解决IPV4本身IP数量少的问题(43亿).所以使用私有网段来解决一个局域网下ip的问题.再结合NAT技术,可以让使用私有网段IP的主机继续正常与网络中的拥有公网IP的主机进行通信.
    Ps:(因此2台手机使用qq是并不是2个手机上的qq直接进行通信,有兴趣可以自行了解)
    所以,IPv4枯竭的问题,最终解决方案是使用IPv6.

  1. 路由控制
    有了全球唯一的ip地址以后,在网络上的每个节点(路由器)借助各自的路由控制表来管理基于ip协议的数据包传输控制.路由控制表的形式有2种:

    1. 静态路由控制,由管理员手动设置

    2. 动态路由控制,当路由器与其他路由器互相交换信息时自动刷新.

  1. 相关衍生概念

    1. DNS
      IP地址类似xxx.xxx.xxx.xxx的字符串,因为非常难以记忆.所以借助DNS(Domain Name System)的支持,可以用便于记忆的域名(比如www.google.com),来访问google网站. DNS会将域名解析为具体的IP,然后再进行通信.

    2. ARP
      在网络层,只要知道了IP地址,便可以向这个目标地址发送IP数据段,
      而在底层的数据链路层中进行实际通信时,是需要了解路由过程中每个IP地址所对应的MAC地址的
      ARP便是解决此问题的协议.ARP借助ARP请求与ARP相应2种类型的包来确定目前IP的MAC地址

      过程:
      主机A为了获得主机B的MAC地址,起初需要通过广播发送一个ARP请求.借助广播机制,最终目标主机B会将自己的IP地址以ARP响应的形式返回给主机A.
      为了减少每次通信都需要先利用ARP协议获取目标MAC地址,主机会缓存将IP与MAC的映射关系缓存到ARP缓存表中一段时间.(减少网络大量广播的情况),也因为IP与MAC地址的关系不是永久有效的,所以也只能缓存一段时间.

    3. PARP
      RARP(Reverse Address Resolution Protocol),是ARP的反协议.通过MAC地址来获得IP地址.

    4. DHCP(Dynamic Host Configuration Protocol)
      以前的主机加入网络前需要管理员手动指定ip地址,每一次设定ip都需要保证ip的唯一性,当ip越来越多以后,会变得十分繁琐.所以DHCP就是为了解决手动分配ip地址的协议.
      为了检查将要分配的IP地址以及已经分配的IP地址是否可用,DHCP服务器或者客户端必须具备以下功能:

      1. DHCP服务器:在分配IP地址前发送ICMP回送请求包,确认没有返回应答.
      2. DHCP客户端,针对从DHCP那里获取的IP地址发送ARP请求包,确认没有返回应答.
        在这个规则下,可用保证安全地进行IP地址自动分配.

    5. NAT
      NAT(Network Address Translator) 是用于在本地网络中使用私有地址,在连接互联网时使用公网IP地址的技术.NAT技术实际是为了解决IPv4地址枯竭而开发的技术.不过,在IPv6种为了提高网络安全也在使用NAT.
      后来还有NAPT技术(可以转换TCP,UDP端口号),由此可以实现用一个IP地址与多个主机的通信.

      1. NAT工作机制:

        在NAT(NAPT)路由器的内部,有一张自动生成的记录转换地址的表,当10.0.0.10向163.221.120.9发送第一个包时生成这个表,并按照表中的映射关系进行处理.上述流程如果有多台机器与外部通信时,仅仅对IP进行转换,可能导致端口冲突(不同主机可能都使用同一端口对外通信),所以NAPT可以解决下该问题.

        上图,主机163.221.120.9的端口是80,LAN中有两个客户端10.0.0.10和10.0.0.11同时进行通信,并且这2个客户端的本地端口都是1025.此时如果仅仅转换IP地址为路由器当前的公网IP地址:202.244.174.37,会导致冲突.
        为此,只要将其中任一导致冲突的IP(10.0.0.11)的端口号转换为空闲的端口(例如1026),就可以解决问题.
        NAT维护的转换表由路由器自动管理,在第一次建立TCP链接时生成,在收到关闭连接时,发出FIN包的确认应答后从表中移除.

      2. NAT的缺陷:
        由于NAT(NAPT)都依赖自己的转换表,会导致以下缺陷:

        1. 无法从NAT的外部向内部服务器建立连接.
        2. 转换表的生成和转换操作都会产生一定的开销
        3. 通信过程中一旦NAT遇到异常重启时,所有的TCP连接都将被重置.

      3. NAT穿越:
        解决外部无法主动向NAT内部主机发起请求的方案有多种,具体可以参考该文章:http://www.52im.net/article-64-1.html
